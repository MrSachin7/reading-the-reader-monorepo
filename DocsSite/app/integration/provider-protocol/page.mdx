# External AI Provider Integration Guide

This guide is for teams that want to connect a real AI decision-maker to the Reading the Reader backend over `/ws/provider`.

<Callout type="info">
  The external AI provider does not talk to the frontend protocol. It talks to the dedicated provider protocol on `/ws/provider`.
</Callout>

## Boundary

- Browser clients use `/ws`
- External AI providers use `/ws/provider`
- The backend remains authoritative for session state, decision state, intervention validation, and intervention application

## Activation

The provider flow is active only when the backend decision configuration is set to:

- `providerId = "external"`
- `executionMode = "advisory"` or `executionMode = "autonomous"`

Example REST request:

```http
PUT /api/experiment-session/decision-configuration HTTP/1.1
Content-Type: application/json

{
  "conditionLabel": "External AI",
  "providerId": "external",
  "executionMode": "advisory",
  "automationPaused": false
}
```

## Current Runtime Flow

```mermaid
sequenceDiagram
    participant Participant
    participant Backend
    participant Provider
    participant Researcher

    Participant->>Backend: viewport / focus / attention updates
    Backend->>Provider: providerViewportChanged
    Backend->>Provider: providerReadingFocusChanged
    Backend->>Provider: providerAttentionSummaryChanged
    Backend->>Provider: providerGazeSample
    Backend->>Provider: providerDecisionContext

    alt advisory mode
        Provider->>Backend: providerSubmitProposal
        Backend->>Researcher: decisionProposalChanged
        Researcher->>Backend: approveDecisionProposal / rejectDecisionProposal
        Backend->>Provider: providerDecisionModeChanged
    else autonomous mode
        Provider->>Backend: providerRequestAutonomousApply
        Backend->>Provider: providerDecisionModeChanged
        Backend->>Provider: providerInterventionEvent
    end
```

## Key Rules

- send only provider messages on `/ws/provider`
- preserve `providerId`, `sessionId`, and `correlationId`
- treat backend module ids and parameter keys as strict contract values
- let the backend remain the final authority for proposal lifecycle and intervention application
